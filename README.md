Camus
=====

使用 C++ 开发的静态博客生成工具，markdown 解析工具采用 cmark，支持一定程度的自定义主题。

## 用法

直接读取当前目录

```shell
camus
```

设置工作目录

```shell
camus [./blog]
```

### 配置

目录结构

```shell
.
├── assets
│     ├── css
│     │     ├── home.css
│     │     └── page.css
│     ├── favicon.ico
│     └── robots.txt
├── camus.yaml
├── html
├── posts
│     └── example.md
└── theme
```

程序启动将会自动加载 ${work_dir}/camus.yaml

```yaml
camus:
   # 源文件夹
   source_dir: posts
   # 输出文件夹
   output_dir: html
   # asserts/* 原样复制到 output_dir/
   assets_dir: assets
   # 自动生成 sitemap.xml
   sitemap: true
   # 加载 theme/name/*.html
   theme: pixel
   # 转换md的输出文件名：keep 不转换，uuid 自动转换为 /uuid.html
   filename_case: keep
   # 目录来源：auto 自动生成到 toc.yaml，但修改该文件无效
   # 其余值则自动尝试读取为 yaml
   toc: auto

site:
   # 主页模板和文章内页模板会自动加载 theme/home.html 和 theme/page.html
   # 主标题，显示在首页的 title 中
   title: Just Another Camus blog
   # 副标题，显示在首页的描述中
   subtitle: Static blog generated by Camus
   # 描述，显示在首页，一般是一段作者信息等文字
   description: Rome was not built in one day.
   # 站点的主要 URL
   url: https://stdout.com.cn
   # 站点的 git repo
   repo: https://github.com/o8x/camus
```

## 文章参数

位于 markdown 文件头部

```yaml
---
short-path: simple1
display-name: 示例1
date: 2025-03-21
visibility: open
---
```

| 参数           | 用途                                                       |
|--------------|----------------------------------------------------------|
| short-path   | 不为空时，优先使用 short-path 作为最终生成文件名                           |
| display-name | 显示名称，即文章标题和目录中显示的标题                                      |
| date         | 文章发布时间                                                   |
| visibility   | 文章可见性，可选 open（默认），hidden（隐藏，不渲染）, hidden-in-toc（不显示在目录中） |

### 隐藏内容

支持在文章中使用 `--hidden-section-start|end` 标记隐藏多行内容

```
...
--hidden-section-start
这里的内容将不会被渲染
--hidden-section-end
...
```
